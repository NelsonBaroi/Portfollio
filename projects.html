A selection of tools I've built while exploring AI, audio, and simulation. Each one started with a specific question — not a technology to showcase. The approaches vary, but the goal was always the same: understand how something actually works by building it.

AI Memory

Local knowledge base with file indexing and interactive Q&A

What I worked on

  • Real-time file watcher that indexes new and modified files automatically
  • Embedding-based semantic search across the indexed knowledge base
  • LLM-powered Q&A that answers questions from retrieved context
  • Threshold tuning for when retrieval should trigger vs. general knowledge

Stack

Python Embeddings Watchdog LLM Q&A
What I learned The more interesting problem wasn't indexing — it was deciding when a search result is relevant enough to use. Retrieval-augmented generation is only as good as the relevance filter in front of it; without that, the model confidently answers from the wrong document.
github.com/NelsonBaroi/AI-Memory

Qwen Voice Cloner

Multi-language voice cloning using Qwen3-TTS

What I worked on

  • Voice cloning pipeline: upload a reference sample, generate matching speech
  • Voice design from text descriptions ("warm, calm, slightly accented")
  • Built-in speaker library and voice sample management
  • CUDA / bfloat16 optimization for GPU inference; CPU fallback
  • 10-language support: English, Russian, Chinese, Japanese, Korean, and more

Stack

Python PyTorch Qwen3-TTS Gradio SoundFile Librosa
What I learned Small errors in transcript alignment — a word cut off, a pause misread — compound into noticeable artifacts in the output. The preprocessing pipeline matters as much as the model. I also found that voice design from text descriptions is harder to control than cloning from a real sample; the model's interpretation of "warm" varies.
github.com/NelsonBaroi/Qwen-Voice-Cloner

Hairstyle Try-On

AI-powered hairstyle editing with FLUX and Qwen cloud models

What I worked on

  • Image intake, face detection, and preprocessing pipeline
  • Dual-engine architecture: FLUX as primary, Qwen as automatic fallback
  • 15+ hair colors with a custom hex picker
  • Gender-specific style catalogs; generation history for comparison
  • Multi-language UI (English and Russian)

Stack

Python Streamlit Pillow OpenCV FLUX API Qwen API
What I learned Designing for model unreliability was the main engineering challenge. Cloud APIs fail intermittently, and if the app doesn't handle that gracefully, the user sees an error rather than a result. Building the fallback path at the UI layer — so the switch from FLUX to Qwen is invisible — made the experience feel stable even when individual calls weren't.
github.com/NelsonBaroi/Hairstyle-Try-On

Parallel Universe Me

Life simulation using Monte Carlo methods and real statistical datasets

What I worked on

  • Monte Carlo simulation engine for probabilistic life-path modeling
  • Causal DAGs mapping decision trees with statistical outcome weights
  • Data compilation from BLS, OECD, World Bank, and Kauffman research
  • LLM narrator that interprets simulation output — without generating numbers
  • Gradio frontend with Plotly visualizations; Vercel serverless deployment

Stack

Python Gradio Plotly Pandas Monte Carlo Vercel
What I learned Keeping the LLM out of the data generation was a deliberate constraint — and the right one. Once a model starts producing statistics, users can't distinguish them from real ones, which undermines the whole point. The LLM's role here is narrative: it explains what the Monte Carlo output means. The numbers come from real datasets. That separation is harder to maintain architecturally than it sounds.
github.com/NelsonBaroi/parallel-universe-me

SFX Generator

Procedural sound effects from signal processing — no samples required

What I worked on

  • 18+ SFX types: impacts, sweeps, drones, glitches, ambient textures, alarms
  • Butterworth filters, amplitude envelopes, LFO modulation from scratch
  • Adjustable duration, frequency, intensity, and stereo pan per effect
  • Batch generation mode; WAV export at 44.1 kHz, 16-bit stereo
  • Tkinter desktop GUI for interactive use

Stack

Python NumPy SciPy SoundFile Tkinter
What I learned This project uses no AI models — everything is signal processing. That turned out to be a useful exercise: designing sounds from frequency curves and filter shapes requires understanding what you actually want the audio to do, rather than describing it in natural language and hoping a model interprets it correctly. Procedural generation gives you complete control, at the cost of needing to know what you're controlling.
github.com/NelsonBaroi/SFX-Generator

Kira

Structured personal knowledge base designed for LLM context injection

What I worked on

  • Organized 19-chapter biography, career materials, and relocation planning
  • Visa queue analytics: Python scripts, weekly CSV datasets, 13 statistical charts
  • Machine-readable manifest.json indexing all files by category and format
  • Exported and structured 6 months of AI conversation history
  • Designed for reuse: any file can be fed as context to any LLM tool

Stack

Python Markdown JSON HTML
What I learned A knowledge base is only useful if you can retrieve the right piece at the right time. The manifest.json — a structured index of every file with metadata — was the detail that made everything else work. Without it, the repository is just a folder of documents. With it, any tool can navigate the contents without reading everything. The real challenge was deciding what belongs in a knowledge base versus what belongs in a conversation.
github.com/NelsonBaroi/Kira
